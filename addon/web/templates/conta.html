Nota sobre o login pelo navegador (ingress). O fluxo OAuth do Google redireciona para localhost, que através do ingress aponta para a máquina errada. Enquanto essa etapa não é resolvida no front, o login é feito uma vez por terminal (headless) — abaixo. O token gerado fica salvo e renova sozinho; você não repete isso.

No host do add-on (ou em qualquer máquina com o projeto), rode:

OAUTH_PORT={{ oauth_port }} python -m src.orquestrador --conta <nome> --login

Ele imprime uma URL. Abra-a (se estiver por SSH, use um túnel: ssh -L {{ oauth_port }}:localhost:{{ oauth_port }} <host>), autorize com a conta Google desejada, e o token.json é criado em config/<nome>/. Recarregue esta página para ver a conta aparecer.
